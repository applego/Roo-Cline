### 2. 現在発生している問題点（WebChatAdapter 関連）

- **型の不一致エラー**

    - **エラー内容:**
        - WebChatAdapter.ts の 52行目で、`{ url: string; token: string; }` 型のオブジェクトが、string 型のパラメーターとして扱われている。
    - **考えられる原因:**
        - WebSocketClient のコンストラクタは、現状 string 型の URL を受け取る実装になっているが、WebChatAdapter.ts 側ではオブジェクト（例: `{ url, token }`）を渡しているため、型が合わなくなっている可能性があります。

- **存在しないメソッドの参照**
    - **エラー内容:**
        - WebChatAdapter.ts の 131行目で、WebSocketClient に存在しない `disconnect` メソッドが呼び出されています（実際の実装では `close` メソッドが定義済み）。
    - **考えられる原因:**
        - 呼び出し側のコードが、WebSocketClient の仕様とずれている可能性が高いです。

## 今後の指示書

以下のステップに沿って、エラー解消および機能改善を進める方針を提案します。

### 1. WebSocketClient の仕様見直し

- **コンストラクタの引数について**
    - **現状:**
        - WebSocketClient は string 型（URL）のみを受け取る実装となっている。
    - **問題点:**
        - WebChatAdapter.ts では、`{ url: string; token: string; }` 型のオブジェクトが渡されている。
    - **対応案:**
        - **案A:** WebSocketClient のコンストラクタをオブジェクト型（例: `WebSocketConfig`）を受け取るように変更する。
            - **メリット:** 拡張性が高く、トークンなどの追加情報も管理可能。
            - **デメリット:** 既存の呼び出し箇所の修正が必要になる可能性がある。
        - **案B:** WebChatAdapter.ts 側で引数を文字列（URL）のみに変更する。
            - **メリット:** WebSocketClient の実装を変更する必要がない。
            - **デメリット:** トークン情報などが不要となるため、将来的な拡張性が低下する可能性がある。

### 2. disconnect メソッド問題の修正

- **現状:**
    - WebSocketClient に `disconnect` メソッドが存在せず、呼び出し側でエラーが発生している（代わりに `close` メソッドが存在）。
- **対応案:**
    - **案A:** WebSocketClient に `disconnect` メソッドを新たに追加し、内部で `close()` を呼び出すように実装する。
        - 例:
            ```ts
            public disconnect(): void {
              this.close();
            }
            ```
    - **案B:** WebChatAdapter.ts 側で `disconnect` メソッドの呼び出しを `close` に置き換える。
