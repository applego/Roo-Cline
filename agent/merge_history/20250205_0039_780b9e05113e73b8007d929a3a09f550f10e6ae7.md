Gemini 2.0 Flash Thinking Experimental 01-21の要約

---

以下の差分を要約します。

**全体的な変更:**

この差分は、プロジェクトを "Roo Cline" から "Roo Code" に名称変更し、多くの新機能や実験的な機能を追加する大幅な拡張を反映しています。コードベース全体にわたって機能拡張とリファクタリングが行われています。

**主な追加機能:**

- **名称変更:** プロジェクト名が "Roo Cline" から "Roo Code" に更新されました。UI、ドキュメント、コード内の名称が更新されています。
- **新機能追加:**
    - **カスタムモード:** カスタムモード機能とその関連コード（設定、UI、テストなど）が追加されました。
    - **コードアクション:** コードアクション機能とその関連コードが実装されました。
    - **Unbound プロバイダー:** Unbound AI プロバイダーのサポートとその関連コードが追加されました。
    - **レート制限:** API リクエストのレート制限機能とその関連コードが実装されました。
    - **実験的機能:** 新しい実験的機能（unified diff 戦略、insert content ツール、search and replace ツール、new task ツール、switch mode ツール）が追加されました。
    - **統合テスト:** 統合テストの仕組みとその関連設定、コードが追加され、テストカバレッジが強化されました。
    - **Storybook & ドキュメント:** Storybook の設定ファイル、ドキュメント関連ファイル (Jekyll サイト) が追加されました。
- **アーキテクチャの拡張:**
    - Webview UI のビルドプロセスが Create React App から Vite ベースの高度な構成に更新されました。
    - サーバーと Webview UI のモジュールシステムが最適化されました。
    - 状態管理と設定管理の仕組みが強化されました。
- **依存関係の更新と拡張:**
    - `package.json` と `pnpm-lock.yaml` が更新され、新機能のための依存関係が追加されました。`npm` から `pnpm` にパッケージマネージャーが更新されました。
    - サーバー側の依存関係に `cors`、`express`、`uuid`、`dotenv` が追加され、テスト関連の devDependencies が強化されました。
    - Webview UI 側に Storybook 関連、Vite 関連の依存関係が追加され、開発環境が強化されました。
- **API プロバイダーの拡張:**
    - AWS Bedrock、DeepSeek、Gemini、OpenAI Native、OpenRouter、VS Code Language Model (Copilot) の各プロバイダーのコードが、新機能追加に合わせて拡張されました。特に、カスタムモデル情報の設定や、DeepSeek Reasoner などの特殊なモデルへの対応が追加されています。
    - Glama プロバイダーは機能強化されました。
    - LM Studio プロバイダーは新機能に対応するよう更新されました。

**画面の拡張:**

- **プロンプト設定画面:** カスタムモードや高度なプロンプト設定に関連する項目が追加され、画面が機能強化されました。
- **設定画面:** 実験的機能の設定項目、モード切り替えの自動承認設定などが追加され、より詳細な設定が可能になりました。
- **チャット入力欄:** プロンプト強調設定のアイコンが追加されました。
- **履歴画面:** エクスポートボタンなど、新機能が追加され、機能が強化されました。
- **ウェルカム画面:** Roo Cline から Roo Code へ名称変更が反映されました。
- **自動承認メニュー:** モード切り替えとタスク作成の自動承認オプションが追加され、メニューが拡張されました。
- **チャット画面:** 推論ブロック (Reasoning Block) など、新しい UI 要素が追加され、表示が強化されました。

**API の拡張:**

- **追加された API:**
    - Unbound プロバイダー関連の API
    - レート制限関連の API
    - カスタムモード関連の API
    - コードアクション関連の API
    - MCP サーバーのタイムアウト設定 API
- **強化された API:**
    - 各プロバイダー API が拡張され、新しい設定項目や処理が追加されました。
    - MCP Manager API が強化され、初期化や設定ロードの処理が最適化されました。
    - WebSocket Server API にレート制限ロジックが追加されました。
    - ConfigManager API の一部メソッド名が最適化され、`resetAllConfigs` メソッドが追加されました。

**全体として、この差分はプロジェクトをより高機能で柔軟な形に拡張することを目的とした大規模な機能追加とリファクタリングであると言えます。** 特に、実験的機能や高度なカスタマイズ機能が追加され、コードベースが大幅に強化されている点が特徴です。UI もそれに合わせて拡張され、高度なチャット機能と詳細な設定機能が追加されています。API の変更も UI の拡張と連動しており、新しい機能をサポートするための API が追加・強化されています。
