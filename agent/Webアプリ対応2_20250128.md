    - 既存のコードベースを再利用可能な形でモジュール化する方針を立てます。
2.  **Webアプリケーション版のアーキテクチャ設計:**
    - フロントエンド (React)、バックエンド (Node.js)、データベース (必要に応じて) の3層構造を基本とし、各層の役割と技術スタックを決定します。
    - フロントエンドとバックエンド間の通信方式 (WebSocket、REST APIなど) を選定します。

    - まずは、コア機能であるプロンプトの送受信とテキストベースの応答表示に焦点を当て、ファイル操作やツール実行などの高度な機能はMVPの範囲外とします。
4.  **開発環境のセットアップ:**
    - フロントエンド、バックエンドそれぞれの開発に必要なツール (Node.js, pnpm, React開発環境など) をセットアップします。
    - TypeScriptを導入し、型安全な開発環境を構築します。
    - コード Linting, Formatting ツール (ESLint, Prettier) を導入し、コード品質を維持します。

**ミニマムバリアブルプロダクト (MVP) 設計**

MVPでは、Webアプリケーション版Roo-Clineの基本的な土台となる部分を構築します。

- **フロントエンド (webview-ui):**
    - ReactとTypeScriptでUIを構築します。
    - ユーザーがプロンプトを入力するテキスト入力欄と、AIからの応答を表示するチャット履歴表示領域を実装します。
    - シンプルなデザインとし、UIフレームワークは styled-components を使用します。
    - 状態管理には、React Context API を使用します (最初はシンプルにし、必要に応じて Zustand や Recoil などのライブラリを検討します)。
    - バックエンドとの通信には WebSocket を使用し、リアルタイムな応答表示を実現します。
- **バックエンド (server):**
    - Node.jsとExpressでバックエンドサーバーを構築します。
    - WebSocketサーバーを実装し、フロントエンドからの接続を待ち受け、メッセージを送受信します。
    - Roo-Clineのコアロジック (`src/core`) をバックエンドに統合し、プロンプト処理を行います。
    - APIエンドポイント `/messages` をWebSocket通信用として定義します。
- **通信:**
    - フロントエンドとバックエンドは WebSocket で接続し、双方向通信を行います。
    - メッセージのフォーマットは JSON を使用し、テキストメッセージの送受信を基本とします。

**詳細設計**

MVP設計をさらに具体化し、詳細な設計を行います。

1.  **フロントエンド (webview-ui)**

    - **技術スタック:** React, TypeScript, styled-components, WebSocket API
    - **ディレクトリ構成:**
        ```
        webview-ui/
        ├── src/
        │   ├── components/       # Reactコンポーネント
        │   │   ├── ChatArea.tsx  # チャット履歴表示領域
        │   │   └── InputArea.tsx # テキスト入力欄、送信ボタン
        │   ├── hooks/          # React Hooks
        │   ├── styles/         # styled-components 用スタイル定義
        │   ├── App.tsx         # メインコンポーネント (チャットUI全体を構成)
        │   ├── index.tsx       # アプリケーションのエントリーポイント
        │   └── ...
        ├── public/             # 静的ファイル (HTML, CSS, 画像など)
        ├── package.json
        ├── pnpm-lock.yaml
        └── tsconfig.json
        ```
    - **コンポーネント:**
        - `App`: チャットUI全体のレイアウトと状態管理を担うメインコンポーネント。`ChatArea` と `InputArea` を配置。
        - `ChatArea`: チャットメッセージの履歴を表示するコンポーネント。メッセージのリストをpropsで受け取り、表示をレンダリング。
        - `InputArea`: テキスト入力欄と送信ボタンを配置するコンポーネント。ユーザーの入力をstateで管理し、送信ボタンクリック時にバックエンドへメッセージを送信する処理を実装。
    - **状態管理:**
        - チャットメッセージの履歴は、`App` コンポーネントの state で管理し、Context API を通じて `ChatArea` コンポーネントに提供します。
    - **通信処理:**
        - `App` コンポーネント内で WebSocket クライアントを初期化し、バックエンドとの接続を確立します。
        - `InputArea` から送信されたメッセージを WebSocket を通じてバックエンドへ送信します。
        - バックエンドから受信したメッセージをチャット履歴に追加し、UIを更新します。

2.  **バックエンド (server)**

    - **技術スタック:** Node.js, Express, ws (WebSocketライブラリ), TypeScript
    - **ディレクトリ構成:**
        ```
        server/
        ├── src/
        │   ├── routes/         # APIルート定義 (最初はWebSocketのみ)
        │   ├── core-integration.ts # Roo-Clineコアとの統合処理
        │   ├── server.ts       # サーバーのエントリーポイント、WebSocketサーバー起動
        │   └── ...
        ├── package.json
        ├── pnpm-lock.yaml
        └── tsconfig.json
        ```
    - **WebSocket API エンドポイント (`/messages`):**
        - クライアントからの WebSocket 接続を受け付けるエンドポイントを `/messages` として定義します。
        - クライアントからテキストメッセージを受信したら、Roo-Cline のコアロジック (`src/core`) を呼び出してプロンプトを処理します。
        - 処理結果 (テキスト応答) を JSON 形式でクライアントに WebSocket を通じて送信します。
    - **Roo-Cline コア統合 (`core-integration.ts`):**
        - `src/core` ディレクトリのコードをバックエンドプロジェクトにコピーし、必要なモジュールをインポートします。
        - コアロジックを呼び出すためのインターフェースを実装します (例: `processPrompt(prompt: string): Promise<string>`)。
        - VS Code API に依存する部分を Web 環境で動作するように修正または代替実装を行います (例: ファイルシステムアクセス、設定管理など)。

3.  **開発ステップ:**
    1.  **バックエンド開発:**
        - `server` ディレクトリを作成し、`pnpm init` で `package.json` を初期化します。
        - Express と `ws` ライブラリをインストールします (`pnpm add express ws`)。
        - TypeScript の設定 (`tsconfig.json`) を行います。
        - `src/server.ts` に WebSocket サーバーの基本構成 (WebSocket 接続待ち受け、メッセージ送受信の基本的なロジック) を実装します。
        - `src/core-integration.ts` を作成し、`src/core` から必要なファイルをコピーして統合します (最初は `Cline.ts` などコアとなるファイルのみ)。
        - `core-integration.ts` 内に、プロンプトを受け取り、コアロジック (`Cline.ts` の `ask` メソッドなど) を呼び出して応答を返す `processPrompt` 関数を実装します。
    2.  **フロントエンド開発:**
        - `webview-ui` ディレクトリを `pnpm create vite webview-ui --template react-ts` などで作成し、React + TypeScript のプロジェクトを初期化します。
        - `src/App.tsx` を修正し、チャットUI の基本レイアウト (`ChatArea`, `InputArea` コンポーネントの配置) を実装します。
        - `src/components` ディレクトリに `ChatArea.tsx` と `InputArea.tsx` コンポーネントを作成し、それぞれのUI要素 (チャット履歴表示、テキスト入力欄、送信ボタン) を実装します。
        - `App.tsx` 内で WebSocket クライアントを初期化し、バックエンド (`ws://localhost:3000/messages` など) に接続します。
        - `InputArea` からメッセージを送信する処理、バックエンドからメッセージを受信して `ChatArea` に表示する処理を実装します。
    3.  **統合テスト:**
        - バックエンドサーバー (`server/src/server.ts`) を起動します (`pnpm dev` など nodemon を使用して開発サーバーを起動)。
        - フロントエンド (`webview-ui`) を起動します (`pnpm dev`)。
        - Webブラウザでフロントエンドアプリケーション (`http://localhost:5173` など) にアクセスし、チャットUI が表示されることを確認します。
        - テキスト入力欄にプロンプトを入力し、送信ボタンをクリックしてバックエンドにメッセージを送信し、応答がチャット履歴に表示されることを確認します。
        - 簡単なプロンプト (例: "Hello", "What is your name?") でエンドツーエンドの疎通テストを行います。
