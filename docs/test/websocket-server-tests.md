# WebSocketサーバーテスト仕様書

## 1. テスト概要

WebSocketサーバーの機能を検証するためのテストスイートです。主にメッセージハンドリング、接続管理、エラー処理の3つの観点からテストを実施します。

## 2. テスト対象コンポーネント

- `WebSocketServerImpl`
- `MessageRouter`
- `ConfigStore`
- `McpManager`

## 3. テストケース

### 3.1 接続管理テスト

#### 3.1.1 新規接続

- 新しいWebSocket接続が確立された時の動作を確認
- 接続IDが正しく生成されることを確認
- connectionHandlerが呼び出されることを確認

#### 3.1.2 接続終了

- クライアントが切断した時の動作を確認
- 接続マップから該当する接続が削除されることを確認

### 3.2 メッセージハンドリングテスト

#### 3.2.1 API設定更新

- `upsertApiConfiguration`メッセージの処理を確認
- ConfigStoreに正しく保存されることを確認
- 更新結果が正しくクライアントに返されることを確認

#### 3.2.2 MCPツール呼び出し

- `use_mcp_tool`メッセージの処理を確認
- McpManagerが正しく呼び出されることを確認
- ツール実行結果が正しくクライアントに返されることを確認

#### 3.2.3 設定取得

- `getConfig`メッセージの処理を確認
- 現在の設定が正しくクライアントに返されることを確認

### 3.3 エラー処理テスト

#### 3.3.1 不正なメッセージフォーマット

- JSON解析できないメッセージを送信した場合のエラー処理を確認
- 適切なエラーメッセージがクライアントに返されることを確認

#### 3.3.2 未知のメッセージタイプ

- 未定義のメッセージタイプを送信した場合のエラー処理を確認
- 適切なエラーメッセージがクライアントに返されることを確認

#### 3.3.3 実行時エラー

- ConfigStore操作時のエラー処理を確認
- McpManager操作時のエラー処理を確認
- エラー情報が正しくクライアントに返されることを確認

## 4. テストカバレッジ目標

- 行カバレッジ: 90%以上
- 分岐カバレッジ: 85%以上
- 関数カバレッジ: 100%

## 5. テスト実行方法

```bash
# すべてのテストを実行
pnpm test

# 特定のテストファイルのみ実行
pnpm test websocket-server.test.ts

# カバレッジレポートを生成
pnpm test:coverage
```

## 6. テスト結果の確認

テスト実行後、以下の場所でカバレッジレポートを確認できます：

- テキスト形式: コンソール出力
- HTML形式: `coverage/lcov-report/index.html`

## 7. 継続的インテグレーション

GitHub Actionsを使用して、以下のタイミングでテストを自動実行します：

- プルリクエスト作成時
- mainブランチへのマージ時
- リリースタグ作成時
